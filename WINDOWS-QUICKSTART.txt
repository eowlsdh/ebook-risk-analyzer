ebook-risk-analyzer Windows 빠른 시작
====================================

이 도구는 EPUB, HTML/XHTML/XML, TXT 전자책의 사람이 검토할 신호를 로컬에서 정리합니다.
결과는 진단이나 판정이 아닙니다. 기본 분석은 네트워크에 연결하지 않습니다.
명령줄 분석과 향후 릴리스의 로컬 브라우저 앱을 사용할 수 있습니다. 이 소스 체크아웃에 Setup.exe나 휴대용 실행 폴더가 이미 포함된 것은 아닙니다.

1. Python 설치
---------------
https://www.python.org/downloads/windows/ 에서 Python 3.11 이상 64비트 설치 관리자를 받습니다.
설치 첫 화면의 "Add python.exe to PATH"를 선택합니다.
설치 후 새 PowerShell에서 확인합니다.

    py -3.11 --version

2. 프로젝트 설치 (PowerShell)
-----------------------------
프로젝트 폴더를 연 뒤 아래를 한 줄씩 복사하여 실행합니다.

    py -3.11 -m venv .venv
    .\.venv\Scripts\Activate.ps1
    python -m pip install .
    ebook-risk-analyzer --help

실행 정책 오류가 나면 현재 PowerShell 창에서만 아래를 실행하고 다시 활성화합니다.

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
    .\.venv\Scripts\Activate.ps1

명령 프롬프트(cmd.exe) 사용 시:

    py -3.11 -m venv .venv
    .venv\Scripts\activate.bat
    python -m pip install .

3. 샘플 분석
------------

    ebook-risk-analyzer analyze .\examples\sample_book --output .\out\sample --language auto --format all

출력 폴더에서 report.html을 열어 사람이 검토합니다. report.json, findings.csv,
extracted_text.txt, metadata.json도 함께 생성됩니다.

4. 내 EPUB 분석
---------------

    ebook-risk-analyzer analyze .\my-book.epub --output .\out\my-book --language ko --format all
TXT 또는 단일 HTML 파일도 직접 지정할 수 있습니다. 예: `ebook-risk-analyzer analyze .\my-book.txt --output .\out\my-book --format all`

옵션: --language auto|ko|en, --config 파일경로, --max-file-size 바이트,
--verbose, --fail-on high|critical|never 를 사용할 수 있습니다.
`--max-file-size`를 넘는 파일이나 디렉터리는 건너뛰지 않고 즉시 오류로 종료합니다. 값을 늘리거나 입력을 분할한 뒤 다시 실행합니다.
--verify-links는 외부 링크 확인을 요청할 수 있으므로 폐쇄망 또는 민감한 원고에서는
지정하지 마십시오.

5. 두 판본 비교
---------------

    ebook-risk-analyzer compare .\drafts\first.epub .\drafts\revised.epub --output .\out\compare --format all

오프라인 설치
-------------
인터넷 연결 컴퓨터에서 프로젝트 폴더에서 실행:

    py -3.11 -m pip wheel --wheel-dir wheelhouse .

wheelhouse 폴더와 프로젝트를 폐쇄망 Windows 컴퓨터로 복사한 뒤:

    py -3.11 -m venv .venv
    .\.venv\Scripts\Activate.ps1
    python -m pip install --no-index --find-links .\wheelhouse .

문제 해결
---------
- 명령을 찾지 못하면 새 터미널을 열고 가상 환경을 활성화한 뒤
  python -m ebook_risk_analyzer --help 를 실행합니다.
- 설치가 인터넷을 찾으면 --no-index와 --find-links .\wheelhouse를 모두 사용했는지
  확인합니다.
- 결과의 발췌문에는 원문 일부가 포함될 수 있습니다. 원고와 결과를 같은 보안 수준으로
  보관하고, 분석 권한이 있는 저작물만 처리합니다.
- finding은 원본 위치와 전후 문맥을 확인해 종결하거나 편집 작업으로 바꾸십시오.
  반복 표현, 번역체, 장르 관습, 의도적 문체는 오탐일 수 있습니다.
6. 로컬 브라우저 앱 (향후 배포물)
---------------------------------
이 절은 게시자가 실제로 제공한 설치형 `Setup.exe` 또는 휴대용 폴더를 받은 경우의 안내입니다.
소스 폴더만 받았다면 2번의 Python 설치를 사용하십시오.

설치형: `EbookRiskAnalyzer-Setup-x64.exe`의 파일 속성에서 발급자/디지털 서명을 확인한 뒤
실행합니다. 기본 설치는 관리자 권한 없이 현재 사용자에게만 `%LocalAppData%\Programs\Ebook Risk Analyzer`에
이루어집니다. 제거는 Windows 설정의 **설치된 앱**에서 합니다.

휴대용: 압축을 푼 폴더 **전체**를 `%LocalAppData%\Apps\EbookRiskAnalyzer` 같은 쓰기 가능한 사용자
로컬 위치에 복사하고, 폴더 안의 실행 파일을 실행합니다. 실행 파일 하나만 옮기면 안 됩니다.
제거하려면 앱을 종료한 뒤 휴대용 폴더를 삭제합니다. 조직 전체 관리 배포는 표준 설치와 별개로
IT 관리자가 승인된 관리 도구로 구성·검증하며, 일반 사용자는 관리자 자격 증명을 제공할 필요가 없습니다.

처음 실행하면 브라우저가 `http://127.0.0.1:포트/`를 엽니다. 이 주소는 같은 PC에서만
사용합니다. Windows 방화벽이 공용/개인 네트워크 허용을 묻는 경우 허용하지 말고
취소합니다. 브라우저 창을 닫아도 앱은 종료되지 않습니다. 화면의 인증된 **앱 종료**를
사용해 종료합니다. 한 번에 하나의 인스턴스만 실행할 수 있습니다. 충돌 뒤 남은 임시 작업
공간은 다음 시작 시 안전하게 정리됩니다. 내려받은 보고서는 별도이므로 직접 삭제합니다.

화면에서 EPUB, HTML/HTM/XHTML/TXT 파일을 끌어 놓거나 **파일 선택**을 누릅니다.
여러 HTML/TXT 원고는 **폴더 선택**으로 올립니다. EPUB은 단독으로 선택합니다.
언어를 고르고 **로컬 분석 시작**을 누릅니다. **외부 링크도 확인하기**는 기본으로
꺼져 있으며, 승인된 네트워크에서 꼭 필요할 때만 켭니다.

완료 화면에서 사람이 원문과 제작 이력을 확인한 뒤 다음 다섯 결과를 내려받습니다.

    report.json
    report.html
    findings.csv
    extracted_text.txt
    metadata.json

결과물에는 원문 발췌와 메타데이터가 포함될 수 있습니다. 원고와 같은 보안 수준으로
보관하고, 필요 없어진 내려받은 보고서는 조직 정책에 따라 삭제합니다. 이 도구는
AI 작성 여부, 저작권 또는 품질을 진단하거나 판정하지 않습니다.

폐쇄망에는 인터넷 연결 PC에서 검증한 설치형 또는 휴대용 **빌드 산출물 전체**를
승인된 매체로 복사합니다. 링크 확인을 켜지 않으면 기본 분석은 네트워크를 사용하지
않습니다. 실행 파일 없이 소스만 배포해야 할 때는 "오프라인 설치"의 wheelhouse와
`--no-index --find-links` 절차를 사용합니다.

앱이 열리지 않으면 표시된 127.0.0.1 주소를 같은 PC 브라우저에 붙여 넣습니다.
다른 PC나 휴대폰에서 접속할 수 없습니다. 파일이 거부되면 허용 확장자·크기를
확인하고, `..`가 든 경로, 바로가기, 심볼릭 링크는 제거합니다. 업데이트는 앱을
종료한 뒤 새 `EbookRiskAnalyzer-Setup-x64.exe`를 실행하거나 휴대용 폴더 전체를 새 폴더로 교체합니다.

게시자 유지보수: 실제 빌드 산출물만 배포하고, 릴리스 전에 실제 `Setup.exe`를 Windows에서
반드시 빌드·시험합니다. 깨끗한 Windows 표준 사용자 계정에서 설치·실행·제거와 휴대용 실행,
127.0.0.1 전용 바인딩, 방화벽 예외 미생성, 기본 네트워크 미사용, 파일/폴더 업로드, 거부 입력,
다섯 다운로드, 업데이트와 제거를 점검합니다. 릴리스 노트에는 실제 파일명·버전·서명·
해시를 기록하며, 바이너리가 이 소스 저장소에 이미 있다고 안내하지 않습니다.
