# Windows 배포 및 유지보수 안내

이 문서는 배포 담당자용입니다. 저장소를 내려받은 것만으로는 설치 프로그램이나 휴대용 폴더가 만들어지지 않습니다. 릴리스 빌드가 만든 미래의 `Setup.exe`와 휴대용 폴더를 검증·배포할 때 사용하십시오.

## 배포물 구분

- **설치형 출력물(향후 빌드):** `dist-installer\EbookRiskAnalyzer-Setup-x64.exe`입니다. 기본 설치는 관리자 권한 없이 현재 사용자만을 위해 `%LocalAppData%\Programs\Ebook Risk Analyzer`에 이루어지며, 시작 메뉴와 선택한 바탕 화면 바로가기 및 Windows **설치된 앱**의 제거 항목도 현재 사용자에게만 만들어집니다.
- **휴대용 출력물(향후 빌드):** 예를 들어 `dist\EbookRiskAnalyzer\` 폴더 전체입니다. 사용자는 폴더 전체를 로컬 드라이브에 복사한 뒤 안의 실행 파일을 실행합니다. 일부 파일만 복사하면 실행되지 않습니다.
- **소스 체크아웃:** `README.md`와 `pyproject.toml`이 있는 개발자용 폴더입니다. 위 두 배포물이 없을 때만 Python 설치 절차로 실행합니다. 소스 폴더를 설치형 배포물처럼 안내하지 마십시오.

현재 이 저장소에는 완성된 바이너리가 포함되어 있다고 주장하지 않습니다. 배포 담당자는 서명·악성코드 검사·대상 Windows에서의 실행 확인을 마친 실제 빌드 산출물만 배포합니다.
## 조직 관리 배포

표준 설치에는 관리자 권한이 필요하지 않습니다. 조직의 모든 사용자에게 관리형으로 배포해야 하면 IT 관리자가 별도의 엔드포인트 관리 정책 또는 승인된 기계별 배포 패키지를 사용하고, 대상 Windows에서 그 배포·업데이트·제거를 검증합니다. 표준 사용자용 `Setup.exe`에 관리자 권한이나 인증서·서명 비밀을 포함하지 않습니다.


## 최종 사용자 안내문

1. 설치형이면 `EbookRiskAnalyzer-Setup-x64.exe`를 내려받아 마우스 오른쪽 단추로 **속성**을 열어 발급자와 디지털 서명을 확인한 뒤 실행합니다. UAC 관리자 자격 증명은 필요하지 않습니다. 휴대용이면 압축을 풀고 폴더 전체를 `%LocalAppData%\Apps\EbookRiskAnalyzer` 같은 쓰기 가능한 사용자 로컬 위치에 둡니다.
2. 처음 실행하면 앱은 브라우저를 열고 `http://127.0.0.1:포트/`로 연결합니다. 이는 이 PC 안에서만 연결되는 주소입니다. Windows 방화벽 창이 나타나면 **개인 네트워크나 공용 네트워크 접근을 허용하지 말고 취소**합니다. 앱은 외부 공개용 서버가 아닙니다.
3. 화면에 EPUB 또는 HTML/HTM/XHTML/TXT 파일을 끌어 놓거나 **파일 선택**을 누릅니다. 여러 장으로 된 원고는 **폴더 선택**을 사용합니다. EPUB은 단독으로 올립니다.
4. 언어를 확인하고, 링크 확인은 기본값인 꺼짐으로 둡니다. 외부 링크를 실제 확인해야 하는 승인된 환경에서만 **외부 링크도 확인하기**를 선택합니다.
5. 분석이 끝나면 브라우저 결과 화면에서 내용을 사람 검토하고 `report.json`, `report.html`, `findings.csv`, `extracted_text.txt`, `metadata.json`을 각각 내려받습니다. 결과에는 원문 발췌와 메타데이터가 있을 수 있습니다.
6. 브라우저 창을 닫아도 앱은 종료되지 않습니다. 화면의 인증된 **앱 종료**로 로컬 서버를 멈춥니다. 한 번에 하나의 인스턴스만 실행할 수 있으며, 충돌 뒤 남은 안전하게 표시된 임시 작업 공간은 다음 시작 시 정리됩니다. 내려받은 보고서는 별도이므로 필요에 따라 직접 삭제합니다.


이 도구는 AI 작성 여부, 저작권, 품질을 진단하거나 판정하지 않습니다. 모든 신호는 원문과 제작 이력을 사람이 확인해야 합니다.

## 개인정보, 폐쇄망, 정리

입력은 앱의 개인 임시 작업 공간에서 처리되며 원본 파일을 직접 내려받는 기능은 없습니다. 앱을 정상 종료하면 이 임시 작업 공간이 정리됩니다. 다만 결과물에는 발췌문이 포함될 수 있으므로 원고와 같은 접근 권한으로 보관합니다. 작업 후에는 내려받은 보고서를 조직 보존 정책에 따라 옮기거나 삭제하고, 휴대용 폴더/설치형 앱을 종료합니다.

폐쇄망에는 인터넷 연결이 가능한 PC에서 검증한 설치형 또는 휴대용 **빌드 산출물 전체**를 승인된 매체로 복사합니다. 기본 분석은 네트워크를 사용하지 않습니다. 링크 확인을 켜지 마십시오. 실행 파일이 없고 소스만 있는 경우에는 프로젝트 루트의 `README.md`에 있는 `wheelhouse` 기반 `--no-index` 설치 절차를 사용합니다.

## 문제 해결, 업데이트, 제거

- 브라우저가 열리지 않으면 앱이 표시한 `http://127.0.0.1:포트/` 주소를 같은 PC의 브라우저에 붙여 넣습니다. 다른 PC나 휴대폰에서 열 수 없으며 열려고 해도 안 됩니다.
- 방화벽 또는 보안 제품이 네트워크 공개를 요구하면 허용하지 말고 취소합니다. `127.0.0.1` 이외 주소로 열리지 않는 배포물을 사용합니다.
- 파일이 거부되면 허용 확장자와 크기를 확인하고, `..`가 들어간 상대 경로·바로가기·심볼릭 링크는 제거합니다.
- 설치형 업데이트는 이전 앱을 종료한 뒤 새 `EbookRiskAnalyzer-Setup-x64.exe`를 실행합니다. 휴대용 업데이트는 이전 폴더와 보고서를 백업하고 새 폴더 전체로 교체합니다.
- 설치형 제거는 Windows **설치된 앱**에서 앱을 선택해 제거합니다. 휴대용 제거는 앱을 종료하고 휴대용 폴더를 삭제합니다. 원고와 내려받은 보고서는 별도 위치에 있으므로 필요하면 따로 삭제합니다.

## 게시자 릴리스 점검

1. 깨끗한 Windows VM에서 설치형과 휴대용 폴더를 각각 설치/실행합니다.
2. `127.0.0.1`만 수신하는지, 방화벽 예외를 만들지 않는지, 기본 분석이 네트워크 요청을 하지 않는지 확인합니다.
3. 허용 파일, 폴더 업로드, 거부 확장자, 경로 탈출, 큰 파일, 결과 다섯 개 내려받기, 종료 후 정리를 점검합니다.
4. 서명 인증서·발급자 표시·버전·업그레이드·제거를 확인하고, 릴리스 노트에 실제 산출물 이름과 해시를 기록합니다.
5. 소스 빌드가 필요한 고객에게는 `README.md`의 Python/wheelhouse 절차를 제공하고, 바이너리가 이미 저장소에 있다는 표현은 사용하지 않습니다.
6. 실제 `Setup.exe`는 릴리스 전에 Windows에서 반드시 빌드하고, 표준 사용자 설치·실행·제거와 휴대용 실행을 모두 시험합니다. 다른 운영체제에서 만든 파일이나 소스 체크아웃을 설치형 산출물로 배포하지 않습니다.

## GitHub Actions에서 Windows 설치 파일 만들기

저장소의 `.github/workflows/windows-build.yml`은 `main` 브랜치 push, `v*` 태그 push 또는 Actions 화면의 수동 실행으로 Windows 빌드를 수행합니다.

1. GitHub 저장소에서 **Actions**를 엽니다.
2. **Build Windows Installer**를 선택합니다.
3. **Run workflow**를 누르고 `main`을 선택합니다.
4. 작업이 성공하면 실행 상세 화면 아래 **Artifacts**에서 다음 두 항목을 내려받습니다.
   - `EbookRiskAnalyzer-Setup-x64`: 설치 프로그램 EXE
   - `EbookRiskAnalyzer-Portable-x64`: 무설치 ZIP
5. ZIP을 풀어 실제 `.exe` 또는 Portable ZIP을 확인하고 SHA-256 해시를 배포 기록에 남깁니다.
6. 실제 Windows 사용자에게 전달하기 전에 깨끗한 Windows 환경에서 설치, 실행, 샘플 분석, 앱 종료, 제거를 확인합니다.

Actions Artifact는 30일 동안 보존됩니다. 현재 저장소가 비공개이면 저장소 접근 권한이 있는 사용자만 워크플로와 산출물을 볼 수 있습니다. 워크플로에는 원고, 분석 보고서, `.gjc` 세션, 로컬 경로, API 키 또는 서명 인증서가 포함되지 않습니다. 코드 서명을 추가할 때는 인증서를 저장소 파일로 커밋하지 말고 GitHub Actions Secret과 조직 승인 절차를 사용하십시오.
